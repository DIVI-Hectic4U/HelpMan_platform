# HelpMan Platform — Complete Testing Walkthrough

This document provides a comprehensive step-by-step guide to testing every feature of the HelpMan platform from end to end.

---

## 1. Initial Setup & Running Locally

Before testing the features, ensure your local development environment is running:

1. **Database Setup**:
   Ensure your `.env` has a valid Neon PostgreSQL `DATABASE_URL`.
   Run the following to initialize the schema and seed test data:
   ```bash
   npx prisma db push
   npx prisma db seed
   ```
2. **Start Servers**:
   Start both the Next.js frontend and Express backend:
   ```bash
   npm run dev
   ```
   *Frontend will run on `http://localhost:3000`*
   *Backend will run on `http://localhost:4000`*

---

## 2. Authentication & Onboarding Flow

**Goal:** Verify user registration, validation, and login.

1. **Register**:
   - Navigate to `http://localhost:3000/register`.
   - Try to enter a password without numbers or uppercase letters — verify the validation error prevents submission.
   - Enter valid details and submit.
   - You should be automatically redirected to the `/dashboard`.
2. **Login**:
   - Log out by clearing your browser local storage (or using a logout button if present).
   - Navigate to `http://localhost:3000/login`.
   - Enter your credentials to ensure JWT authentication succeeds.

---

## 3. Account Linking (Settings)

**Goal:** Verify users can link their competitive programming profiles.

1. Navigate to the **Settings** page via the dashboard sidebar.
2. Under "Linked Accounts", enter a Codeforces handle (e.g., `tourist`) and a LeetCode handle.
3. Save your preferences. 
4. Check the PostgreSQL database (via `npx prisma studio`) to ensure the `User` table reflects your updated handles.

---

## 4. Telegram Bot & AI Task Generation

**Goal:** Verify the Express backend correctly handles Telegram webhook events and calls Gemini AI.

Since Telegram webhooks require a public URL, you have two options to test this locally:

**Option A (Using Postman to simulate Telegram):**
1. Send a `POST` request to `http://localhost:4000/api/whatsapp/webhook`.
2. Provide this mock JSON payload to simulate a user typing `/daily`:
   ```json
   {
     "message": {
       "text": "/daily",
       "chat": {
         "id": 123456789
       }
     }
   }
   ```
3. Check the backend console logs. You should see it fetching your linked CF/LC profile, querying Gemini AI, and returning 3 tailored JSON problems!

**Option B (Using Ngrok & Real Telegram):**
1. Run `ngrok http 4000`.
2. Set your Telegram Bot Webhook URL to `https://<your-ngrok-url>.ngrok.app/api/whatsapp/webhook`.
3. Open Telegram and send `/daily` to your bot.
4. You should instantly receive a personalized message with 3 coding problems.

---

## 5. Gamification (XP, Streaks, and Ranks)

**Goal:** Verify the streak multiplier algorithm and rank promotion system.

1. **Complete a Task**:
   - Simulate completing a task by sending `/done 1` via Postman or Telegram to the bot webhook.
   - Verify the bot replies with XP awarded (e.g., "+35 XP (Easy)").
2. **Verify Streak Multiplier**:
   - Open your database using `npx prisma studio`.
   - Change your user's `streak` column to `7`.
   - Send `/done 2` via the webhook.
   - The XP awarded should be multiplied by ~1.97x (e.g., 50 XP becomes 98 XP).
3. **Verify Rank Promotions**:
   - In the database, manually set your user's XP to `1990` (Silver rank).
   - Complete another task (`/done 3`).
   - You should receive an extra **Rank Up Bonus** (+150 XP) and visually graduate to **Gold 🥇**.
4. **View XP History**:
   - Open the web dashboard.
   - Verify the line chart and Recent Activity feed displays your new XP transactions accurately.

---

## 6. Dashboard & Leaderboard

**Goal:** Verify data synchronization between the backend API and Next.js frontend.

1. **Dashboard Home** (`/dashboard`):
   - Check that the progress bar accurately reflects your XP relative to the next rank threshold (e.g., 2000 XP for Gold).
   - Verify the "Today's Tasks" section shows the 3 problems generated by Gemini.
2. **Leaderboard** (`/dashboard/leaderboard`):
   - Verify your user appears on the leaderboard.
   - Check the top-3 podium visuals (Bronze, Silver, Gold colors).

---

## 7. Admin Panel Security

**Goal:** Ensure unauthorized users cannot access system metrics.

1. Navigate to `http://localhost:3000/dashboard/admin`.
2. If you are a standard user, the backend API requests (to `/api/admin/stats`) should return a `403 Forbidden` error (visible in browser Network tab), and the panel should show a permission denied state.
3. Open the database (`npx prisma studio`), change your user's `role` from `USER` to `ADMIN`.
4. Refresh the page. You should now see full system metrics (Total Users, API usage, Rank Distributions).
